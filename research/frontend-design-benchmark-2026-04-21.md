## Purpose

This memo captures the strongest current front-end conclusions after:

- local competitive research
- screenshot-aware Anthropic review of the current homepage
- screenshot-aware Anthropic review of the current team page
- a broader Anthropic strategy pass on how to compete with real sports products

## Bottom line

The project already has:

- strong data ideas
- differentiated analytical structure
- real product depth

But the current front-end still reads more like:

- a smart prototype
- an engineer-built research tool

than:

- a premium daily-use sports destination

The data feels more premium than the presentation.

That gap is now the biggest front-end problem.

## The actual design target

The site should not try to become:

- a budget ESPN clone
- a generic sports portal
- a prettier spreadsheet

The correct design target is:

- `premium fan anthropology lab`
- `cinematic team dossier`
- `editorial front page for argument-loving CFB fans`

The site should feel like:

- a place that knows the sport emotionally
- a place with strong taste
- a place where each chart exists because it tells a story

not:

- a neutral dashboard with extra rows

## What real sports sites still do better

Current mainstream products still win on:

- immediate visual confidence
- trust signals
- emotional first-screen impact
- team identity expression
- mobile scanability
- sense of freshness and movement

Even when their underlying product is less interesting, they often feel more:

- official
- alive
- daily-usable

That is what we need to close.

## What the site already does well

The current product already has meaningful advantages:

- cross-division comparison is genuinely distinctive
- rating movement storytelling is a real moat
- historical resemblance / reminiscence ideas are memorable
- the mood / fan-intelligence direction can be much more original than standard sports pages

This means:

- we do not need better commodity widgets
- we need better packaging of the differentiated stuff we already own

## Biggest visual weaknesses right now

These are the recurring themes from the reviews.

### 1. Not enough visual energy

The pages feel too muted and too gray.

That makes them read as:

- respectable
- organized

but also:

- bloodless
- anonymous

College football pages need more emotional texture than that.

### 2. Weak first-screen punch

The hero area needs to feel like:

- a destination
- a story opening
- a reason to keep scrolling

The current first screens are more informative than memorable.

### 3. Mobile density is too punishing

This is the biggest concrete product problem.

The current layouts ask mobile users to:

- parse too much at once
- read too many similarly weighted blocks
- work too hard to find the main story

### 4. Flat hierarchy

Too many elements feel similarly important.

That weakens:

- scanning
- perceived polish
- narrative clarity

### 5. Weak trust and authority cues

Major sites benefit from:

- familiar branding
- timestamps
- bylines
- source attribution
- clear update cadence

This project needs its own version of those trust signals.

## Design direction to commit to

### Homepage

The homepage should feel like:

- an editorial front page

not:

- a giant board with everything weighted equally

Best homepage priorities:

- `Biggest Vibe Shifts`
- `Respect Gap Leaders`
- `Main Character Of The Week`
- `Cross-Division Shock Rankings`
- `Rival Heat Leaders`

That creates a homepage with:

- motion
- narrative
- argument fuel
- visual variety

### Team page

The team page should feel like:

- a living dossier

The first screen should likely center on:

- major record/rank context
- a flagship `Team Mood Card`
- a strong season-story visual

The page should answer quickly:

- `how good are we?`
- `how do our fans feel?`
- `how are outsiders talking about us?`
- `what changed?`

### Matchup page

The matchup page should not be a generic betting matrix.

It should feel like:

- an argument theater

Best ingredients:

- `Market vs Mood`
- `What Both Sides Fear`
- `Which Fanbase Is Calmer`
- `Model vs Market vs Mood`

## Visual system recommendations

### 1. Let team identity show up harder

Use:

- team colors
- logos / wordmarks
- subtle branded color washes
- clearer conference / level color coding

Do not let the site become a chaotic rainbow.

But do let teams feel like themselves.

### 2. Stronger type hierarchy

The biggest numbers and strongest story points should dominate.

Examples:

- rank
- record
- power rating
- belief status
- weekly delta

These should not compete visually with metadata or body copy.

### 3. More editorial spacing rhythm

Pages need:

- clearer section breaks
- more breathing room
- better large/small alternation

This matters as much as color.

### 4. Progressive disclosure everywhere

Especially on mobile:

- show story first
- let detail expand later

Do not shrink dense desktop tables and call that a mobile layout.

### 5. Atmosphere without clutter

Use:

- gradients
- subtle texture
- soft depth
- occasional image-driven hero moments

Avoid:

- flat monotony
- busy wallpaper
- generic card soup

## Product components that should feel premium

These are the surfaces that can become signature brand objects.

### Team Mood Card

This should be:

- visual
- shareable
- glanceable
- emotionally legible

It should not read like:

- six analytics KPIs in a gray box

### Rating River / Season Journey

This is one of the most ownable visuals in the product.

It should be treated like:

- hero evidence

not:

- a secondary chart

### Cross-Division Context

This is a major moat and should feel intuitive.

Always pair national rank with:

- division context
- nearby comparables
- visual grouping by level

### Weekly editorial cards

Cards like:

- `Most Panicked Fanbases`
- `Biggest Vibe Shifts`
- `Living Rent-Free`

should look like:

- magazine callouts
- social-friendly story cards

not:

- rows in a sortable admin table

## What not to do

These are the wrong directions.

### Do not chase generic portal features

- score widgets
- commodity news blocks
- generic injury centers
- shallow recruiting content

Those make the product broader and weaker.

### Do not make everything a table

Tables are useful.

But if the product becomes:

- table after table after table

it will never feel premium.

### Do not obsess over theme toggles

Dark mode is not the answer to visual differentiation.

The actual questions are:

- hierarchy
- identity
- emotional energy
- scanability

### Do not present uncertainty only in footnotes

Confidence should become part of the design language:

- faded states
- dotted rules
- softer color treatment
- confidence labels

## Highest-leverage next front-end changes

1. redesign the homepage around editorial blocks instead of one flat board
2. make the hero area more emotionally legible and visually bold
3. rebuild mobile layouts with card-first progressive disclosure
4. strengthen the type scale and spacing rhythm across all pages
5. add more trust signals:
   bylines, update timestamps, source notes, methodology entry points
6. make team identity and conference / level identity more visually present
7. turn flagship modules into visually distinct signature objects

## Final design test

If the site feels like:

- `smart`
- `complete`

but not:

- `alive`
- `confident`
- `memorable`

then the design is still unfinished.

The right end state is:

- serious enough to trust
- stylish enough to prefer
- fun enough to revisit

That is the bar if the site wants to stand favorably against real sports products.
