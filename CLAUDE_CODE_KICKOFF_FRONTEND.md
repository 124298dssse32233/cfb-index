# Claude Code Kickoff — Frontend Migration (Python Generator + Alpine + v5 Tokens)

**Context**: CFB Index ships as a Python-generated static site (17,426 HTML pages). The v5 design system lives in a Figma Make React prototype. This kickoff brings v5 into production without rewriting the generator. The output is: one external stylesheet on OKLCH tokens, Alpine.js for interactivity, 10 modules ported from React reference to Python + Alpine, dark-mode default, zero visual regressions on the live site during any stage.

**Outputs**:

   ```
   <article class="the-room" data-module="the-room" x-data="theRoom({{ cohort_data_json }}, '{{ default_cohort }}')">
     <!-- cohort pills -->
     <!-- belief dial -->
     <!-- trajectory spark (SVG; x-bind path d to derived data) -->
     <!-- top-quote blockquote -->
     <!-- confidence + sample strip -->
   </article>
   ```
2. Alpine component registered in output/site/assets/js/the-room.js:
   ```js
   document.addEventListener('alpine:init', () => {
     Alpine.data('theRoom', (buckets, initial) => ({
       cohort: initial,
       cohorts: buckets,
       get active() { return this.cohorts[this.cohort]; },
       selectCohort(id) {
         this.cohort = id;
         window.urlState.set('room', id);  // URL sync helper
       },
     }));
   });
   ```
3. URL sync helper at output/site/assets/js/url-state.js:
   ```js
   window.urlState = {
     get(key) { return new URLSearchParams(location.search).get(key); },
     set(key, value) {
       const params = new URLSearchParams(location.search);
       if (value == null || value === '') params.delete(key);
       else params.set(key, value);
       const url = location.pathname + (params.toString() ? '?' + params : '');
       history.pushState({}, '', url);  // pushState, not replaceState
     },
   };
   ```
4. Hydrate-from-URL on load: `x-data="theRoom(..., window.urlState.get('room') || 'own')"`.
5. CSS for .the-room in cfb-index.css under @layer components.
6. Progressive-enhancement discipline: WITHOUT Alpine (JS disabled), the
   module still renders the 'own' cohort's data statically, no broken UI.
   This means the initial server-rendered HTML MUST contain the default
   cohort's content visibly; Alpine takes over via x-text/x-show at
   hydration.
