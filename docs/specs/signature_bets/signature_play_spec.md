# Signature Play — algorithm spec (S3.2 / §4 Bet #8)

**Purpose.** For each player, surface THE single moment that best
encodes their identity — not the highlight-reel TD, the one that
statistically most represents "what he does."

## Data reality (2026-04-24)

The `plays` table carries 1.1M rows with EPA, PPA, success_flag, and
garbage-time tags — but no play → player_id attribution. That means a
play-level signature is impossible until ingestion lands the bridge
table.

`player_game_stats` carries 281k rows but today is populated only
through 2025 Week 1 — so "Signature Game" (per-game version) is also
coverage-limited.

**Ship path**: build the compute + renderer now with an honest empty
state that dominates on most pages. When ingestion lands, the module
lights up automatically without code changes.

## Signature Score (v1 — game-level)

```
score = stat_value_num × opp_strength_weight × home_away_factor
```

Where:
- `stat_value_num` is the player's total on the signature-story
  headline stat for that game (passing_yds for QBs, rushing_yds for
  RBs, receiving_yds for WRs).
- `opp_strength_weight`: 1.0 vs a top-25 opponent, 0.9 for FBS mid,
  0.6 for FCS / G5. Defaults to 1.0 when team-strength data missing.
- `home_away_factor`: 1.05 on road, 1.0 neutral, 1.0 home. Road games
  score marginally higher so travel-adjusted signature moments surface.

This is intentionally simple. V2 adds EPA × situational weight × play-
profile fit once play-level attribution lands.

## Candidate selection

For each player:
1. Look up the player's signature-metric from the Signature Story
   evaluation (reuse `winner.metric`).
2. Pull every game row where the player has that metric.
3. Drop games where the metric value is None or 0.
4. Score each candidate; pick max.

If coverage < 2 games the module renders empty state — calling one
game "the signature" is dishonest.

## Rendering

Per-card surface:

```
Signature Moment · 2025 Week N vs {Opponent}
Passing yards 342 · result W 34-27 at home
"The pressure-EPA peak of the year against a top-15 havoc defense."
```

The one-line editorial gloss is generated by a small template keyed
off the stat type + opponent tier. It's defensible — never
"insane/electric/generational" copy.

## Cache

`player_signature_plays` table caches one row per (player, season)
with the selected game's identity + score. Build-site computes from
scratch cheaply until the player pool grows; can be backgrounded
later.

```
CREATE TABLE IF NOT EXISTS player_signature_plays (
    player_id      INTEGER NOT NULL,
    season_year    INTEGER NOT NULL,
    game_id        INTEGER NOT NULL,
    week           INTEGER,
    metric_id      TEXT,
    stat_value     REAL,
    score          REAL,
    opponent_name  TEXT,
    home_away      TEXT,
    result_label   TEXT,
    gloss          TEXT,
    generated_at   TEXT DEFAULT (strftime('%Y-%m-%dT%H:%M:%fZ','now')),
    PRIMARY KEY (player_id, season_year)
);
```

## What this spec deliberately does NOT include

- **Play-level attribution** — requires a play_players bridge.
- **Broadcast still frames** — licensing punt; plain text in v1.
- **Season vs per-game ranking** — v1 picks one per (player, season).

## Acceptance criteria

- `python manage.py compute-signature-plays --season 2025` runs.
- Players with ≥ 2 games of coverage render a Signature Moment card
  under the Signature Story section.
- Players with < 2 games (most of the roster today) render empty
  state: "Signature moment lights up once multi-game coverage loads."
