# Page IA Archetypes

**Locked 2026-05-17 (v2-addendum Sprint v5-5.5)**

Every page on CFB Index is one of six archetypes. New pages MUST inherit from an existing archetype unless explicitly approved as a new archetype. The constraint is what makes the site feel like a designed publication rather than a collection of templates.

When an engineer asks "what should this page look like?" the first answer is "which archetype?"

---

## The six archetypes

### 1. Article archetype

**Used by:** `/daily/`, `/mailbag/`, `/reactions/<slug>`, `/editions/<n>/<slug>`

**Structure (top to bottom):**
```
┌──────────────────────────────────────────────────────────────┐
│ Top zone (44px sticky): brand · back · share                  │
├──────────────────────────────────────────────────────────────┤
│ Hero zone (30-40% viewport):                                  │
│   eyebrow (uppercase, 11px UI)                                │
│   headline (clamp 28-48px, serif)                             │
│   dek (clamp 16-22px, serif italic)                           │
│   byline strip (author · date · reading-time chip)            │
├──────────────────────────────────────────────────────────────┤
│ Auto-summary box (highlight color, optional):                 │
│   "30-second read" — 2-3 sentence summary                     │
│   (generated by Pattern A LLM, cached)                        │
├──────────────────────────────────────────────────────────────┤
│ Body (single column, max 70ch, 17-18px Source Serif):         │
│   Pull quotes interrupt the prose (full-width, serif italic)  │
│   Receipt citations as superscript inline                     │
├──────────────────────────────────────────────────────────────┤
│ Footer:                                                       │
│   Citations list (per receipt-pattern doc)                    │
│   Related editions                                            │
│   "Updated" timestamp                                         │
├──────────────────────────────────────────────────────────────┤
│ Bottom sticky strip (44px, mobile only):                      │
│   prev edition · share · related issues                       │
└──────────────────────────────────────────────────────────────┘
```

**Visual mirror:** The Athletic columnist articles + NYT Upshot data essays.

**Allowed:**
- Single column max 680px width on desktop, full-width single column on mobile
- One hero image OR typographic-only treatment (per Visual System Concept)
- Pull quotes, inline charts (per chart vocabulary), in-line citations
- Auto-summary highlight box (1-2 per article)

**Forbidden:**
- Multi-column layouts (use Dashboard archetype if you need them)
- Right-rail "related articles" widgets
- Mid-article ads
- Comment threads (not the moat)
- Auto-playing embeds

### 2. Dashboard archetype

**Used by:** `/`, `/hub/`, `/heisman/`, `/rankings/`

**Structure (top to bottom):**
```
┌──────────────────────────────────────────────────────────────┐
│ Top zone (44px): brand · "what this is" eyebrow               │
├──────────────────────────────────────────────────────────────┤
│ Saturday Strip (in-season) OR countdown strip (off-season):   │
│   32-40px sticky horizontal element                           │
├──────────────────────────────────────────────────────────────┤
│ Hero finding (1 viewport):                                    │
│   eyebrow (11px uppercase)                                    │
│   number (clamp 36-64px, display font)                        │
│   sentence (clamp 14-22px, serif)                             │
│   caption (12px UI, with sample-size chip)                    │
├──────────────────────────────────────────────────────────────┤
│ Primary visualization (full-width annotated chart):           │
│   Per chart vocabulary (no pie, no bar, no radar)             │
│   Annotations IN the chart, not in caption below              │
├──────────────────────────────────────────────────────────────┤
│ Movers grid (2-col mobile, 3-4-col desktop):                  │
│   Each card: 1 tap target, 1 metric, 1 sentence               │
├──────────────────────────────────────────────────────────────┤
│ Drill-down modules (single column on mobile, grid on desktop) │
├──────────────────────────────────────────────────────────────┤
│ Methodology footer:                                           │
│   "How we measure this →" link                                │
│   Sample-size summary                                         │
│   "Updated" timestamp                                         │
├──────────────────────────────────────────────────────────────┤
│ Bottom filter strip (thumb-zone, mobile):                     │
│   Filter chips, opens full-screen modal on tap                │
└──────────────────────────────────────────────────────────────┘
```

**Visual mirror:** Bloomberg Markets dashboard + FiveThirtyEight election dashboard.

**Allowed:**
- Multi-column grids
- Multiple charts per page
- Interactive filters (faceted browse)
- Drill-down to deeper pages
- Sticky top elements (Saturday Strip, brand strip)

**Forbidden:**
- Long-form prose (>200 words consecutive — use Article archetype)
- More than 3 column layouts (overload)
- Decorative imagery (charts only)
- Auto-rotating carousels

### 3. Profile archetype

**Used by:** `/programs/<slug>.html`, `/players/<slug>.html`, `/coaches/<slug>.html`, `/conferences/<slug>.html`

**Structure (top to bottom):**
```
┌──────────────────────────────────────────────────────────────┐
│ Sticky top zone (44px): identity strip (logo + name + key #) │
├──────────────────────────────────────────────────────────────┤
│ Hero zone (30%):                                              │
│   identity (name + key meta in display font)                  │
│   hero number + 1-sentence finding                            │
│   accent tint (team color at 6-8% opacity)                    │
├──────────────────────────────────────────────────────────────┤
│ Modular zones (each ~1 viewport height):                      │
│   Per existing TEAM_PAGE_WORLD_CLASS_BRIEF / PLAYER_PAGE      │
│   State-aware composition (post-loss-sunday, rivalry-week-…)  │
│   Rituals strip (NEW per v5-8.5)                              │
│   Pulse / Chronicle / Savant / Season Arc / Rivalry / etc.    │
├──────────────────────────────────────────────────────────────┤
│ Related-entities strip:                                       │
│   rivalry teams, division opponents, similar teams            │
│   quick-nav cards                                             │
├──────────────────────────────────────────────────────────────┤
│ Footer:                                                       │
│   data depth + methodology + last-updated                     │
└──────────────────────────────────────────────────────────────┘
```

**Visual mirror:** The Athletic columnist-page-per-writer pattern (shared design, per-author variation).

**Allowed:**
- Per-entity bespoke renderer (Sprint v5-9 for teams; v5-10a for players)
- State-aware module promotion/demotion (per docs/design-system/20-page-compositions.md)
- Sticky identity strip
- Multi-section rich layouts
- Rituals strip (teams only, per profile frontmatter)

**Forbidden:**
- Identical layout across entities (every Tier S team page should feel distinct)
- Dashboard-density grids in the hero zone
- Long-form essays in the middle (link out to article archetype)

### 4. Database archetype

**Used by:** `/wire/`, `/editions/` (archive), `/canon/<list>`, `/players/` (directory)

**Structure (top to bottom):**
```
┌──────────────────────────────────────────────────────────────┐
│ Top zone (44px): brand + filter trigger (one-tap)             │
├──────────────────────────────────────────────────────────────┤
│ Header zone:                                                  │
│   page title + 1-sentence "what this is"                      │
│   total count chip ("14 entries · last 30 days")              │
├──────────────────────────────────────────────────────────────┤
│ Active filter chips (dismissible)                             │
├──────────────────────────────────────────────────────────────┤
│ List zone (majority of viewport):                             │
│   Variable-row weight by impact magnitude                     │
│   Big impact = big row with detail                            │
│   Small impact = compact row                                  │
│   Tap any row → drill into detail                             │
├──────────────────────────────────────────────────────────────┤
│ Bottom thumb strip (mobile): sort | filter | new-this-week    │
└──────────────────────────────────────────────────────────────┘
```

**Visual mirror:** Bloomberg terminal feed (variable-row weight) + NY Magazine print archive (magazine-cover wall for editions archive).

**Allowed:**
- Variable row heights by significance
- Faceted filtering
- Sortable columns
- Expandable row details
- Magazine-cover-wall treatment (editions archive specifically)

**Forbidden:**
- Pagination above-the-fold (use infinite scroll or "load more")
- Comment counts (not the moat)
- Visual noise per row (one badge max, one icon max)

### 5. Tentpole archetype

**Used by:** 9 marquee editions per year ("50 Days to Kickoff", SEC Media Days, Big Ten Media Days, Preseason AP Poll, "25 Days", "10 Days", Week 0 Kickoff, Week 1 Hero, etc.)

**Structure:**
```
┌──────────────────────────────────────────────────────────────┐
│ Hero cover (full viewport on first load):                     │
│   Manual illustration treatment (commissioned or hand-       │
│   designed per Visual System Concept)                         │
│   Date prominent (e.g., "AUGUST 29 · WEEK 1")                 │
│   Issue title in display font, max size                       │
│   No chrome                                                   │
├──────────────────────────────────────────────────────────────┤
│ Scroll-trigger transition into Article archetype body         │
└──────────────────────────────────────────────────────────────┘
```

**Visual mirror:** New Yorker covers (illustration-led, no chrome) + NY Mag tentpole issues.

**Allowed:**
- Full-bleed imagery
- Custom typography treatment per edition
- Pattern D adversarial-critique editorial
- Scrollytelling within the body
- One bespoke chart designed specifically for this edition

**Forbidden:**
- Tentpole treatment on non-tentpole editions (rarity is the point)
- Auto-generated cover imagery (manual ChatGPT Plus ritual per the plan)
- Standard Article archetype hero (would dilute the moment)

---

### 6. Anniversary archetype

**Used by:** `/anniversary/today/`, `/saturdays-past/<date>/`

**Structure:**
```
┌──────────────────────────────────────────────────────────────┐
│ Date-anchored eyebrow:                                        │
│   "10 YEARS AGO TODAY · MAY 17, 2016"                         │
├──────────────────────────────────────────────────────────────┤
│ Hero zone:                                                    │
│   The event headline (e.g., "Texas A&M 29, Alabama 24")       │
│   1-sentence cultural context                                 │
├──────────────────────────────────────────────────────────────┤
│ Year-comparison strip:                                        │
│   This date across multiple years (2014, 2018, 2021, etc.)    │
│   Each year clickable to drill into its event                 │
├──────────────────────────────────────────────────────────────┤
│ Body: Pattern C retrospective + "what changed since"          │
├──────────────────────────────────────────────────────────────┤
│ Related-anniversary strip                                     │
└──────────────────────────────────────────────────────────────┘
```

**Visual mirror:** Wikipedia "On this day" + The Athletic retrospective columns.

**Allowed:**
- Date as primary navigation axis
- Year-comparison strips (small multiples)
- Historical photography (when available)
- Archived data from 12-season backfill

**Forbidden:**
- Live data signals (this is retrospective, not current)
- Recommendation engines ("you might also like…")

## How to choose the right archetype

Decision tree:

1. **Is the page a single editorial piece by an author/AI?** → Article
2. **Is the page a live data view that updates frequently?** → Dashboard
3. **Is the page about a single entity (team/player/coach/conference)?** → Profile
4. **Is the page a list/feed of entries that filter and sort?** → Database
5. **Is the page one of the 9 marquee tentpole editions?** → Tentpole
6. **Is the page date-anchored historical retrospective?** → Anniversary

If none fit cleanly, flag in COORDINATION.md for human review before creating a new archetype.

## Mobile behavior per archetype

All archetypes must render correctly at 390px (iPhone Pro), 768px (tablet), and 1280px (desktop). Archetype-specific mobile patterns:

- **Article:** Single column max, sticky bottom strip with prev/share/related, auto-summary box at top
- **Dashboard:** 2-col movers grid (not 4), filter chips in thumb-zone bottom, hero finding sized for one-hand reading
- **Profile:** Sticky identity strip persists; rituals strip horizontal scroll with snap; modules stack vertically
- **Database:** Variable-row weight maintained; filter trigger opens full-screen modal (not dropdown)
- **Tentpole:** Full-bleed hero scales; scroll transition into body works on touch
- **Anniversary:** Year-comparison strip horizontal scroll on mobile

## Performance budgets per archetype

Per `IMPLEMENTATION_PLAN_v2_addendum.md`:
- FCP < 1.5s on 4G
- LCP < 2.5s on 4G
- CLS = 0
- INP < 200ms
- JS < 50KB compressed
- Critical CSS inlined < 10KB

Articles and Profile pages should be on the lighter end (target FCP < 1.0s). Dashboard pages have more chart-rendering cost (target FCP < 1.5s). Tentpole pages are allowed slightly more weight due to hero imagery (target FCP < 2.0s).

Lighthouse CI enforces budgets; fails below score 95.

## Accessibility per archetype

All archetypes commit to WCAG 2.2 AA. Article archetype additionally commits to AAA (long-form reading deserves the higher bar).

Per-archetype specifics:
- **Article:** AAA contrast (7:1 body text), large-print readable, ARIA landmarks for skim navigation
- **Dashboard:** AA contrast on chart elements, table-fallback toggleable for every chart, alt-text describes the finding
- **Profile:** Sticky identity strip not announced repeatedly by screen reader (aria-hidden after first announcement)
- **Database:** Filter and sort controls keyboard-navigable, current filter state announced by aria-live
- **Tentpole:** Hero imagery has descriptive alt-text or `role="presentation"` if decorative
- **Anniversary:** Date math (e.g., "10 years ago") computed at build time, never via JS that screen readers might miss

## Updating this spec

When proposing a new page that doesn't fit cleanly into an archetype:

1. Try harder to fit. The constraint is the value.
2. If genuinely new, write a proposal in COORDINATION.md including: what page, what existing archetype is closest, what's missing, why a new archetype is justified
3. Wait for human review before creating
4. If approved, update THIS file with the new archetype before any rendering code

Page IA decisions are load-bearing. Drift here = inconsistent site = lost "designed publication" feel.
