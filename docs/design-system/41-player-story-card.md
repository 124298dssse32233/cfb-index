# 41 — Player Story Card ("Dossier Noir")

_Status: DESIGN SPEC (critique-hardened v2). Created 2026-06-11 via octopus UI/UX skill + Codex/Gemini adversarial review. Not yet implemented._

A top-of-page narrative module for every player page that frames the player as a character in an ongoing serialized story — generated by the local LLM (Chronicle pipeline) from structured DB rows, regenerated on cadence, grounded in receipts. Mental model from the design-intelligence search: **an OTT / Netflix title page** (poster + recap + "continue the arc"), not a stats table.

Lives in the Noir fan sub-brand ([[40-noir-subbrand]]). Sits ABOVE the existing player v2 modules; it is the "lead," and the modules below are the "receipts" that prove it.

---

## 1. Design principles

1. **Calm at rest, story on intent.** Collapsed by default. A stats-only fan must get real value (identity + key numbers) without reading or expanding anything, and reach the stats in one tap.
2. **One number does the heavy lifting** (Spotify-Wrapped "BAN") — but it must be *honest* (credibility gate, §6) and *labeled* (never a bare number).
3. **Every claim wears its receipt.** Each dramatic statement carries a hoverable/tappable source marker to its DB table. This is the trust moat.
4. **Tier is texture.** The left rail color signals the player's Chronicle tier (S/T1/T2/T3). The page advertises the player's stature before a word is read.
5. **Motion only where it aids orientation** (Linear/Pudding restraint). The only thing that moves is the expand and the chevron.
6. **Never manufacture drama.** Below a data-confidence floor, the dramatic card is suppressed entirely in favor of a factual bio strip (§5). The LLM must not turn missing data into invented arcs.

## 2. Color tokens (dark surface — WCAG-checked)

```css
--psc-ink-800:#16161C;   /* card surface */
--psc-hairline:#2A2A33;       /* structural dividers */
--psc-hairline-strong:#3A3A45;/* control borders */

--psc-gold-deep:#CA8A04; /* foil fills only, never text */

--psc-text-body:#DAD4C9; /* serif prose         (~10:1) */
--psc-text-mut:#ABA59B;  /* meta, timestamps, receipts (~6.5:1 — AA even at small) */

```

Tier rail: S `--psc-gold` · T1 silver `#C7CBD1` · T2 bronze `#B08D57` · T3 flat `--psc-hairline`.

**Contrast floor:** no text pair below 4.5:1 (normal) / 3:1 (≥24px or ≥19px-bold). Muted text never below 12px. Gold-as-graphic (dots, rail) meets 3:1.

**Light/print:** dark-only is deliberate (Noir). Provide `@media print` ink-saver (white bg, near-black ink, gold→`#8a6d10`) and honor `prefers-contrast: more` (bump borders to `--psc-hairline-strong`, text to hi).

No logline, no BAN, no chapter. Just bio + one real stat. Honest > dramatic.

## 6. BAN-selection logic (the heart of the feature)

Pick the single most *surprising-yet-honest* true number. Candidate pool = percentile leaders, rare achievements, perception↔production gaps, records. Score each:

```
score = surprise × rarity × recency × honesty_gate
  surprise = |percentile - 50| normalized, or the aura perception-production delta
  rarity   = 1 - achievement.rarity_pct (rarer = higher)
  recency  = decay over weeks since the stat's season/week
  honesty_gate = 0 if cohort_size < MIN_COHORT (e.g. 30)
                 0 if the slice is gerrymandered (too many qualifiers stacked)
                 0 if sample (plays/carries/games) < MIN_SAMPLE
                 else 1
```

`honesty_gate = 0` ⇒ the candidate is disqualified, not down-weighted. Never ship a cherry-pick. If no candidate clears the gate, the card has no BAN (drop the block, don't fabricate one).

> **Units note:** `achievement.rarity_pct` in `player_achievements` is a pool **percentage** (0–100, = unlocks ÷ eligible × 100), so the code computes `rarity = 1 − rarity_pct/100` — NOT `1 − rarity_pct`.

### 6.1 Selection — tier-priority over flat score (refined 2026-06-12, IMPLEMENTED)

A pure flat `surprise × rarity × recency` across the whole pool **inverts the brand**: the candidate families use non-comparable scales, so recruiting *pedigree* out-scored proprietary *production* (WEPA), and a vanilla counting total out-scored a leaderboard rank — every card monopolized on whichever register happened to score highest in the abstract (the original symptom: every BAN was the perception gap). The fix: each candidate carries a **priority tier**; selection takes the **highest-priority tier present, then the top in-tier score**. The §6 formula now ranks *within* a tier; the `honesty_gate` still disqualifies absolutely. This makes the hierarchy *structural* so the BAN cannot monopolize one register.

| Tier | Register | Candidate sources (implemented) |
|---|---|---|
| 1 · NATIONAL | rarest, most legible national distinction | leaderboard rank (`money_efficiency` #N YPA) · `dual_threat` · **extreme** aura gap (≥25 pctl) |
| 2 · PRODUCTION | the proprietary moat + notable gaps + records | **WEPA** · moderate aura gap (12–25) · `mirror_elite` · a **strong streak** (≥5 straight 100-yd games) |
| 3 · VOLUME | national counting-stat milestone | `volume_king` (cleared the yardage bar) · a **short streak** (3–4) |
| 4 · PEDIGREE | recruiting rank — the honest hero **only for unproven players** | `player_recruiting_profiles` |
| 5 · TEAM | last resort | `program_benchmark` (team-relative leader) |

- **"Records" (the §6 pool) is implemented as the Streak** — the longest run of consecutive 100-yd rushing/receiving games (RB / WR-TE), verified against `player_game_stats`. A QB passing-TD streak is near-universal (most starters throw one every game) and is excluded — it is not a distinction.
- **"Rare achievements" = the numeric `player_achievements`** (leaderboard ranks + volume thresholds). **Honors badges are excluded** (categorical — no honest single number; they live in the selector grid / Trophy Case).
- **NIL valuation is NOT a candidate** — the modeled On3 estimate is never a hero number (owner rule; see [[60-player-page-master]] §6). The NIL *story from discourse* is allowed, the modeled number is not.
- **Implementation + honesty gates:** `story_card._select_ban` (+ `_achievement_ban`, `_streak_ban`); pinned by `tests/test_player_ban_selection.py`. The selected BAN feeds `story_content_hash`, so a flipped BAN correctly re-grounds cached prose ([[59-player-evidence-packet-contract]]).

## 7. Chapter rule

Chapters = career inflection events, in order:
`Recruit commit` → `Season N` (one chapter each) → `Transfer` → `Draft / departure`.
Chapter number = count of inflections to date. Label = LLM-named from the dominant arc of the *current* chapter, governed by the Chronicle banlist (no melodrama, no manufactured stakes).

**Logline stability rule:** the core logline persists across regenerations and only changes when a new inflection event fires. Weekly sentiment swings update the *tension line* and *BAN*, never the logline. Prevents week-to-week whiplash.

---

## 8. Interaction & a11y

- Expand = native `<details><summary>` baseline (works with zero JS / SSR). JS enhances with `grid-template-rows: 0fr→1fr` animation (intrinsic height, no magic number). Collapsed content is `hidden`/not focusable so screen readers don't read it.
- `<summary>` (or a real `<button aria-expanded aria-controls>`): full-width ≥44px target, visible 2px gold focus ring.
- "Jump to stats" is an in-page anchor, ≥44px, visually distinct from the story toggle (unequal goals get unequal weight).
- Motion respects `prefers-reduced-motion: reduce` (instant toggle, no transition).
- Share = copy deep link + (v2) render a share-card PNG for X/Twitter where the CFB community lives.
- AI disclosure: small persistent "AI narrative · updated <date> · sources · report an error" — provenance + freshness + corrections path.

## 8.5 Motion system (disciplined)

**Budget rule** (from UX intelligence — "Excessive Motion" is HIGH severity): **1–2 animated elements per view, maximum.** Parallax / scroll-storytelling is banned (performance + a11y both rated POOR). Micro-interactions (feedback) are cheap and unrationed; "view" animations are rationed hard.

**The statement:** one numeric hero per view, everything else still. The stat chips stay crisp and static — that *makes them read as fact*. The BAN is the single element that animates — that *makes it read as hero*. Static vs. animated does semantic work.

**Tokens**
```css
--psc-ease-out:cubic-bezier(.22,.61,.36,1);   /* entrances, expand */
--psc-ease-in:cubic-bezier(.55,.06,.68,.19);  /* collapses, exits */
--psc-ease-spring:cubic-bezier(.34,1.56,.64,1);/* BAN settle (slight overshoot) */
--psc-t-micro:120ms;   /* hover / press feedback */
--psc-t-state:320ms;   /* expand panel, chevron */
--psc-t-entrance:450ms;/* card fade-up */
--psc-t-ban:420ms;     /* rank settle; magnitude count-up up to 900ms */
```

**Choreography**
- **Collapsed entrance** (on load / IntersectionObserver, plays once): (1) the card fades + `translateY(8px→0)` as ONE unit (450ms ease-out); (2) the BAN is the single numeric hero — *magnitude* numbers count up (≤900ms), *rank* numbers spring-settle (`scale(1.1→1)`, 420ms), landing ~150–200ms after the card. The S-tier gold rail runs a **one-time** top→bottom sweep tied to the same gesture. Identity, chips, tension are STATIC.
- **Expand** (on intent): reveal panel `grid-template-rows:0fr→1fr` (320ms ease-out); inner content fades up as ONE block, not per-paragraph. Chevron rotates 180° (120ms micro). Collapse = ease-in, 240ms.
- **Micro-interactions** (50–120ms, transform/opacity only): stat-chip hover lift + border brighten · `src` marker underline · button `:active scale(.98)` · icon-button color.
- **No ambient loops** (battery + distraction). The "foil" is a one-time entrance sweep, never a continuous shimmer.
- **`prefers-reduced-motion: reduce`** → kill entrance, BAN, sweep, reveal-fade; all state changes snap instantly. The low-data fallback card (§5) is intentionally **motionless** in every case.

**Performance:** animate only `transform` / `opacity` (plus the accepted `grid-rows` technique). Never animate `height`, `color`, `box-shadow`, or `width`. `will-change:transform` on the card during entrance only, cleared after.
